scuff-rf Command-Line Reference


The following table summarizes all command-line options currently available in scuff-rf.

As is true for all programs in the scuff-em suite, command-line options may be specified in a text file catted to standard input; see here for an example of how this works.

Note that, if you find yourself needing more flexibility than can be achieved with these command-line options, then you have probably graduated from using prefab command-line programs to writing your own applications using the c++, python, or matlab interfaces to libscuff.

Option Description
Options describing the geometry
--geometry MyGeometry.scuffgeo Specifies the .scuffgeo file that describes the geometry you are simulating. This option is always mandatory.

For details on how to write a .scuffgeo file, see the reference material here, or look at some examples here and here.

--portfile MyGeometry.ports Specifies the .ports file that defines the ports through which currents are injected into your structure.

For details on how to write a .ports file, see the reference material here, or look at some examples here and here.

Options for requesting port visualization output
--PlotPorts Specifying this option will cause scuff-rf to emit a gmsh visualization file describing the ports in your structure and then exit immediately without performing any calculations. This is useful for confirming that scuff-rf processed your .ports file in the way that you intended.

Running scuff-rf with --PlotPorts will produce a gmsh post-processing file named MyGeometry.ports.pp. Opening this file in gmsh will give you a graphical depiction of all port edges in your structure as interpreted by scuff-rf. (Positive and negative port edges are plotted with values +1 and -1, respectively.) You can use the "merge" command in gmsh (File->Merge) to overlay your full .msh file on top of the .ports.pp file to see how the ports align with the structure.

If --PlotPorts is present on the command line, all command-line options other than --geometry and --ports are ignored, and scuff-rf will terminate immediately upon producing the .ports.pp file.

Options for network parameter mode
--SParameters
--ZParameters
Request computation of S-parameters and/or impedance parameters. You may specify both, either, or neither of these options.
--frequency 3.24

--freqfile MyFreqFile

--minfreq 0.92
--maxfreq 1.23
--numfreqs 10
--logfreq
Specify the frequencies at which network parameters are to be computed. Note that frequencies in scuff-rf are always specified in units of GHz.

The --frequency option requests calculation at a single frequency.

The --freqfile option requests calculation at all frequencies listed in the specified file. (This should be a simple text file with one frequency per line.)

Use --minfreq AA --maxfreq BB --numfreqs CC to request calculations at CC frequencies between AA and BB (including the endpoints). Frequencies will be linearly spaced unless you say --logfreq, in which case frequencies will be logarithmically spaced.

Options for field profile mode
--portcurrentfile MyPortCurrents.portcurrents Specify the file defining the driving port currents.

For details on how to write a .portcurrents file, see the reference material here, or look at some examples here and here.

--EPFile MyEvalPoints.evalpoints Specify the file defining the coordinates of the points at which you wish to evaluate the scattered fields.

For details on how to write an .evalpoints file, see the reference material here, or look at some examples here and here.
